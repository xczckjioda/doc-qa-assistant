An enhanced Retrieval-Augmented Generation (RAG) document QA system with multi-document knowledge base support, hybrid retrieval, Cross-Encoder reranking, query rewriting, citation metadata, related-question suggestions, and RAGAS evaluation.

## Project Overview

`doc-qa-assistant` is an enhanced RAG question-answering system for PDF documents. It supports a multi-document knowledge base backed by ChromaDB and provides a complete QA workflow from upload, chunking, embedding, retrieval, reranking, answer generation, and evaluation.

The core highlights are:

- Hybrid Search: vector retrieval + BM25 keyword retrieval + RRF fusion.
- Rerank: Cross-Encoder reranking before answer generation.
- RAGAS Evaluation: built-in evaluation workflow for faithfulness, answer relevancy, context recall, and context precision.

## Features

- Multi-document support with file-level metadata.
- Append-mode upload: uploading a new file keeps existing documents.
- Same-file replacement: uploading the same `file_name` deletes old chunks for that file before reindexing.
- Hybrid retrieval with vector search, BM25, and Reciprocal Rank Fusion (RRF).
- Query Rewrite while preserving original-query retrieval results for comparison and recall.
- Query Rewrite with Self-Reflection: rewritten queries are automatically quality-checked; invalid rewrites retry up to 2 times, fall back to the original query if still invalid, and expose `rewrite_attempts` in the API response.
- Cross-Encoder reranking with `cross-encoder/ms-marco-MiniLM-L-6-v2`.
- Citation metadata with `file_name` and `page`.
- Related-question suggestions generated by Gemini.
- RAGAS evaluation API and frontend Evaluate tab.
- Single-file frontend using HTML, Tailwind CSS CDN, and native JavaScript.
- Upload, Ask, and Evaluate tabs in the frontend.

## Tech Stack

- Backend: FastAPI, Pydantic, Uvicorn
- PDF parsing: PyMuPDF
- Vector database: ChromaDB
- Embeddings: sentence-transformers, `all-MiniLM-L6-v2`
- Keyword retrieval: rank-bm25
- Reranking: sentence-transformers CrossEncoder, `cross-encoder/ms-marco-MiniLM-L-6-v2`
- LLM: Gemini via Google GenAI SDK
- Evaluation: RAGAS, LangChain Google GenAI, LangChain HuggingFace
- Frontend: single-file HTML, Tailwind CSS CDN, native JavaScript

## Getting Started

### 1. Create a virtual environment

```powershell
cd doc-qa-assistant
python -m venv backend\venv
backend\venv\Scripts\Activate.ps1
```

### 2. Install dependencies

```powershell
cd backend
pip install -r requirements.txt
```

### 3. Configure environment variables

Create `backend/.env`:

```env
GEMINI_API_KEY=your_gemini_api_key_here
LLM_MODEL_NAME=gemini-2.5-flash
```

`LLM_MODEL_NAME` is optional. If omitted, the backend defaults to `gemini-2.5-flash`.

### 4. Start the backend

```powershell
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 5. Open the frontend

Open this file in your browser:

```text
frontend/index.html
```

The frontend calls the backend at:

```text
http://localhost:8000
```

## API Reference

### POST `/upload`

Upload and index a PDF file.

Request:

```http
Content-Type: multipart/form-data
```

Form field:

```text
file: PDF file
```

Response fields:

```json
{
  "filename": "example.pdf",
  "message": "Upload successful",
  "mode": "append",
  "pages_count": 10,
  "chunks_count": 42,
  "preview": []
}
```

Notes:

- `mode` is `append` for a new file.
- `mode` is `replace` when the uploaded `file_name` already exists.
- After upload, BM25 is rebuilt from all chunks in Chroma.

### POST `/ask`

Ask a question against all documents or a selected file.

Request:

```json
{
  "query": "What is the document about?",
  "top_k": 5,
  "file_name": "example.pdf"
}
```

Fields:

- `query`: required user question.
- `top_k`: optional, currently the answer pipeline returns reranked top 5 chunks.
- `file_name`: optional. If provided, retrieval is restricted to chunks from that file.

Response fields:

```json
{
  "query": "What is the document about?",
  "rewritten_query": "document main topic summary",
  "answer": "The document is about ...",
  "sources": [
    {
      "file_name": "example.pdf",
      "page": 1
    }
  ],
  "results": [
    {
      "chunk_id": "example.pdf_page_1_chunk_0",
      "text": "Retrieved chunk text...",
      "file_name": "example.pdf",
      "page": 1,
      "distance": null,
      "rerank_score": 7.1234
    }
  ],
  "suggested_questions": [
    "What evidence supports this answer?",
    "Which section discusses this topic?",
    "Can you summarize the related pages?"
  ]
}
```

### GET `/files`

Return all unique `file_name` values currently stored in Chroma.

Response:

```json
{
  "files": [
    "example.pdf",
    "research_plan.pdf"
  ]
}
```

### POST `/evaluate`

Run RAGAS evaluation for multiple question-answer test cases.

Request:

```json
[
  {
    "question": "What is the research question?",
    "ground_truth": "The research question is ..."
  },
  {
    "question": "What method is used?",
    "ground_truth": "The method is ..."
  }
]
```

Success response:

```json
{
  "success": true,
  "faithfulness": 0.91,
  "answer_relevancy": 0.88,
  "context_recall": 0.83,
  "context_precision": 0.79,
  "metrics": {
    "faithfulness": 0.91,
    "answer_relevancy": 0.88,
    "context_recall": 0.83,
    "context_precision": 0.79
  },
  "results": []
}
```

Failure response:

```json
{
  "success": false,
  "error": "Evaluate failed: ...",
  "faithfulness": null,
  "answer_relevancy": null,
  "context_recall": null,
  "context_precision": null,
  "metrics": {},
  "results": []
}
```

## RAGAS Evaluation

The project includes a RAGAS evaluation module and a frontend Evaluate tab. The evaluation flow calls the local `RetrievalService.retrieve()` directly in the same FastAPI process. It does not start a separate service and does not call `/ask` through HTTP.

Note: `ground_truth` wording should stay close to the original document text; otherwise, Context Recall can be artificially low. Observed project result: faithfulness 1.0 / answer_relevancy 0.886 / context_recall 1.0 / context_precision 1.0.

The four supported metrics are:

- Faithfulness: checks whether the generated answer is supported by the retrieved contexts. It helps detect hallucinations.
- Answer Relevancy: checks whether the answer directly addresses the user question. It helps detect vague or off-topic answers.
- Context Recall: checks whether the retrieved contexts contain the information needed to answer the question. It helps detect missing evidence.
- Context Precision: checks whether the retrieved contexts are relevant and not overly noisy. It helps detect poor retrieval ranking or irrelevant chunks.

To use the frontend evaluator:

1. Open `frontend/index.html`.
2. Switch to the `Evaluate` tab.
3. Add one or more rows with `question` and `ground_truth`.
4. Click `Run evaluation`.
5. Review metric progress bars.
6. Click `Export JSON` to download the evaluation result.

You can also use the CLI script:

```powershell
cd backend
python evaluation\ragas_eval.py
```

The CLI reads:

```text
backend/evaluation/eval_dataset.json
```

and writes:

```text
backend/evaluation/eval_results.json
```

File responsibilities:

- `backend/app/main.py`: FastAPI application entrypoint, route registration, and CORS setup.
- `backend/app/core/config.py`: environment variable loading and runtime settings.
- `backend/app/prompts/qa_prompt.py`: prompt template for answer generation.
- `backend/app/routes/upload.py`: PDF upload, text extraction, chunking, embedding, Chroma write, and BM25 rebuild.
- `backend/app/routes/ask.py`: question-answer endpoint with query rewrite, retrieval, rerank, answer generation, and suggested questions.
- `backend/app/routes/files.py`: lists unique uploaded file names from Chroma metadata.
- `backend/app/routes/evaluate.py`: RAGAS evaluation API endpoint.
- `backend/app/schemas/query_schema.py`: Pydantic models for ask request and response.
- `backend/app/schemas/source_schema.py`: Pydantic model for source metadata.
- `backend/app/services/file_service.py`: extracts PDF text page by page using PyMuPDF.
- `backend/app/services/chunk_service.py`: cleans extracted text and splits it into chunks.
- `backend/app/services/embedding_service.py`: creates embeddings with `all-MiniLM-L6-v2`.
- `backend/app/services/vector_store_service.py`: wraps ChromaDB operations.
- `backend/app/services/bm25_service.py`: builds and searches the BM25 index from Chroma chunks.
- `backend/app/services/hybrid_search_service.py`: combines vector retrieval and BM25 retrieval with RRF.
- `backend/app/services/rerank_service.py`: reranks candidate chunks with a Cross-Encoder.
- `backend/app/services/llm_service.py`: calls Gemini for answer generation, query rewriting, and related-question suggestions.
- `backend/app/services/retrieval_service.py`: orchestrates query rewrite, hybrid retrieval, RRF fusion, noise filtering, rerank, and final answer generation.
- `backend/app/services/qa_service.py`: reserved placeholder for future QA orchestration.
- `backend/evaluation/eval_dataset.json`: sample RAGAS evaluation dataset.
- `backend/evaluation/ragas_eval.py`: CLI/API reusable RAGAS evaluation logic.
- `frontend/index.html`: single-file Tailwind CSS frontend for upload, ask, and evaluation workflows.
